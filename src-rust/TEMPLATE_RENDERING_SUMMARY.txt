IMPLEMENTATION DETAILS
================================================================================

Template Syntax:
  {{variable}}          - Simple variable substitution
  {{nested.path}}       - Nested object access with dot notation
  Multiple occurrences  - Each {{var}} is independently substituted
  Missing variables     - Left as {{var}} in output (safe fallback)

Context Building (in list_resources):
  {
    "uri": resource.uri,
    "name": resource.name,
    "description": resource.description,
    "mimeType": resource.mime_type
  }

Rendering Process:
  1. Scan template for {{ and }} delimiters
  2. Extract variable name between delimiters
  3. Look up value in context (supports nested paths)
  4. Convert value to string representation
  5. Replace {{var}} with rendered value
  6. Continue until entire template processed

Type Conversions:
  - String: used as-is
  - Number: converted to string representation
  - Boolean: "true" or "false"
  - Array: comma-separated values
  - Object: replaced with "[object]"
  - null: "null"

================================================================================
INTEGRATION FLOW
================================================================================

User Code:
  client.list_resources().await?
          ↓
MCPClient::list_resources():
  1. Call MCP server "resources/list"
  2. Get ListResourcesResult with Vec<McpResource>
  3. For each resource:
     a. Check for annotations field
     b. Check for annotations.prompt string
     c. Build context from resource properties
     d. Call TemplateRenderer::render()
     e. Replace resource.description with rendered output
  4. Return modified resources
          ↓
User Code (has templated descriptions):
  Resource {
    description: "Rendered template with substitutions"
  }
